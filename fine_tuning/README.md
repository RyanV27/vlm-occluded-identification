# Fine-tuning

Notebooks and scripts for LoRA fine-tuning vision-language models with [Unsloth](https://unsloth.ai/docs/basics/vision-fine-tuning), on either COCO images or the real-world tabletop dataset.

## Files

| Model | COCO | Real-world tabletop |
|---|---|---|
| Qwen3-VL | `finetune_coco_Qwen3.py` (script only, no notebook) | `Unsloth_Real_Qwen3_VL.ipynb` |
| Qwen2.5-VL | `Unsloth_COCO_Qwen2.5_VL.ipynb` | `Unsloth_Real_Qwen2.5_VL.ipynb` |

- **Real-world tabletop**: 1,000 images used for fine-tuning.

## Setup

Requires a CUDA GPU. Install dependencies (the first code cell in each notebook, commented out by default):

```bash
pip install sentencepiece protobuf "datasets==4.3.0" "huggingface_hub>=0.34.0" hf_transfer
pip install --no-deps unsloth_zoo bitsandbytes accelerate xformers peft trl triton unsloth
pip install transformers==4.57.1
pip install --no-deps trl==0.22.2
```

`xformers` version depends on your installed `torch` version — see the commented install cell in any notebook for the version map. `torch` and `Pillow` are also required and assumed to already be installed.

## Variables to set

Each notebook/script exposes three variables near the top that must be set before running:

```python
# Pretrained model name
PRETRAINED_MODEL = "unsloth/gemma-3-12b-it-unsloth-bnb-4bit"

# Path to fine-tuning data
DATA_DIR = "/scratch/rsvargh2/coco/occluded_images/noise"

# Path to save fine-tuned model checkpoints
MODEL_SAVE_PATH = "/scratch/rsvargh2/finetuned_huggingface_models/unsloth/coco/gemma-3-12b-it-sft-lora-1/"
```

## Expected data format (`DATA_DIR`)

The COCO and real-world notebooks expect different directory layouts and `metadata.json` schemas.

**COCO** — `DATA_DIR` contains three subfolders, each with its own `metadata.json`:

```
DATA_DIR/
├── low/metadata.json
├── medium/metadata.json
└── high/metadata.json
```

Each entry needs `categories_in_image` (list of object labels) and `file_path` (path to the image, resolved as `DATA_DIR/<last two path segments>`). All three severities are concatenated, shuffled, and truncated to the first 10,000 samples for training.

**Real-world tabletop** — `DATA_DIR` contains a single `metadata.json`:

```
DATA_DIR/metadata.json
```

Each entry needs `no_occlusion_objects`, `low_occlusion_objects`, `medium_occlusion_objects`, `high_occlusion_objects` (lists of object labels, concatenated into one answer) and `image_path` (path relative to `DATA_DIR`).

## Running

1. Set the three variables above.
2. Run the notebook top to bottom (or run the `.py` script directly for Qwen3-VL on COCO). Each one: loads the base model in 4-bit, attaches LoRA adapters, builds the conversation dataset from `DATA_DIR`, trains with `SFTTrainer`, then saves the LoRA adapters and tokenizer to `MODEL_SAVE_PATH`.
3. To resume training from a checkpoint or reload a saved model for inference, see the commented-out lines near the bottom of each file (`resume_from_checkpoint=True`, reloading from `MODEL_SAVE_PATH`).

Note: fine-tuned adapters are saved via `save_pretrained`/`push_to_hub`, which saves only the LoRA adapters, not the full model.


## Generating COCO occluded scenes for fine-tuning

```
python generate_occluded_scenes_noise.py --level low --mode noise --count 5000 --action write
python generate_occluded_scenes_noise.py --level high --mode texture --texture path/to/texture.jpg --count 3 --action append
```

Requires the COCO train2017/val2017 images and instance/caption annotations to be downloaded locally (see the docstring in `generate_occluded_scenes_noise.py` for the expected folder structure).

The images generated by this script are also used as training data for finetuning the VLMs on COCO-based synthetic occlusion data.
